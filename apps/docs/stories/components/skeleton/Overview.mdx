## Best Practices

- **Match the content** — skeleton dimensions should approximate the real
  content so the layout doesn't jump when it loads (prevents CLS).
- **Pick the smallest set** — every layout in Patterns is one or two
  primitives. Don't reach for a third unless it earns its keep.
- **Accessibility** — primitives emit `role="progressbar"` and `aria-busy`
  out of the box; no extra ARIA needed for typical use.
- **Animation effect** — set `effect` on the outermost wrapper or on each
  primitive. Keep them consistent within a single loading view.
