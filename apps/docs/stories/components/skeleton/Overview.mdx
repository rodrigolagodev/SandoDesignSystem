<div className="quick-start">
  <h2>Quick Start</h2>

<h3>1. Simple Primitive</h3>
<p>Use the base Skeleton component with shape and dimensions:</p>

```html
<!-- Rectangle skeleton -->
<sando-skeleton width="200px" height="20px"></sando-skeleton>

<!-- Circle skeleton (avatar) -->
<sando-skeleton shape="circle" width="48px" height="48px"></sando-skeleton>

<!-- With animation effect -->
<sando-skeleton effect="shimmer" width="100%" height="16px"></sando-skeleton>
```

<h3>2. Pre-composed Preset</h3>
<p>Drop in a preset for common patterns:</p>

```html
<!-- Card skeleton with image, title, and description -->
<sando-skeleton-card></sando-skeleton-card>

<!-- Comment skeleton with avatar and text lines -->
<sando-skeleton-comment></sando-skeleton-comment>

<!-- Paragraph with multiple lines -->
<sando-skeleton-paragraph lines="4"></sando-skeleton-paragraph>
```

<h3>3. Custom Composition with Layout</h3>
<p>Build custom layouts using layout helpers:</p>

```html
<sando-skeleton-composer>
  <sando-skeleton-row gap="md">
    <sando-skeleton-avatar size="lg"></sando-skeleton-avatar>
    <sando-skeleton-stack gap="sm" style="flex: 1;">
      <sando-skeleton-text size="lg" width="60%"></sando-skeleton-text>
      <sando-skeleton-text size="sm" width="40%"></sando-skeleton-text>
    </sando-skeleton-stack>
  </sando-skeleton-row>
</sando-skeleton-composer>
```

</div>

## Best Practices

- **Match the content** — Skeleton shapes should approximate the actual content dimensions
- **Consistent animation** — Wrap multiple skeletons in `SkeletonComposer` for synchronized effects
- **Accessibility** — Skeletons include `aria-busy="true"` and `role="progressbar"` by default
- **Reduce layout shift** — Use the same dimensions as your actual content to prevent CLS
