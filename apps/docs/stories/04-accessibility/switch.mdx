# Accessibility — sando-switch

> Source: [`packages/components/src/components/switch/ACCESSIBILITY.md`](https://github.com/rodrigolagodev/SandoDesignSystem/blob/master/packages/components/src/components/switch/ACCESSIBILITY.md)

## ARIA pattern

Follows the [WAI-ARIA Switch](https://www.w3.org/WAI/ARIA/apg/patterns/switch/) pattern.

<table>
  <thead>
    <tr>
      <th>Attribute</th>
      <th>Value</th>
      <th>Purpose</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`role`</td>
      <td>`switch`</td>
      <td>Distinguishes a toggle switch from `role="checkbox"`</td>
    </tr>
    <tr>
      <td>`aria-checked`</td>
      <td>`"true"` / `"false"`</td>
      <td>Communicates the on/off state</td>
    </tr>
    <tr>
      <td>`aria-invalid`</td>
      <td>`"true"` / `"false"`</td>
      <td>Marks invalid state when `error=true`</td>
    </tr>
    <tr>
      <td>`aria-describedby`</td>
      <td>Description element ID</td>
      <td>Links helper or error text to the switch</td>
    </tr>
  </tbody>
</table>

Uses a native `<input type="checkbox" role="switch">`. Track and thumb are decorative (`role="presentation"`).

## Screen reader behavior

Off: "Enable notifications, switch, off". On: "Enable notifications, switch, on". Error: "This setting is required, invalid entry".

## Color contrast

Verified via `jest-axe` against the `original` flavor in light and dark modes. Disabled state uses intentional reduced contrast.

## Tested with

- jest-axe (automated, runs in `sando-switch.a11y.test.ts`)
- VoiceOver (macOS Sequoia, Safari 18)
- NVDA (Windows 11, Firefox 130)

## Known limitations

None identified.
