# Accessibility — sando-icon

> Source: [`packages/components/src/components/icon/ACCESSIBILITY.md`](https://github.com/rodrigolagodev/SandoDesignSystem/blob/master/packages/components/src/components/icon/ACCESSIBILITY.md)

## ARIA pattern

Follows the [WAI-ARIA Image](https://www.w3.org/WAI/ARIA/apg/patterns/) pattern — either `img` (semantic) or `presentation` (decorative).

<table>
  <thead>
    <tr>
      <th>Attribute</th>
      <th>Value</th>
      <th>Purpose</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`role`</td>
      <td>`img` (default) / `presentation` (when `decorative=true`)</td>
      <td>Semantic: exposed as image; Decorative: hidden from AT</td>
    </tr>
    <tr>
      <td>`aria-label`</td>
      <td>Icon name or custom label</td>
      <td>Provides the accessible name for semantic icons</td>
    </tr>
    <tr>
      <td>`aria-hidden`</td>
      <td>`"true"` (when `decorative=true`)</td>
      <td>Completely hides decorative icons from AT</td>
    </tr>
  </tbody>
</table>

Use `decorative` attribute when the icon is accompanied by visible text. Provide `aria-label` for semantic icons without text context.

## Screen reader behavior

Semantic icon: "trash, image" (VoiceOver) or "trash graphic" (NVDA). Custom label: "Delete item, image". Decorative icon (`decorative` attribute): not announced.

## Color contrast

Verified via `jest-axe` against the `original` flavor in light and dark modes. All pairs meet WCAG 2.1 AA.

## Tested with

- jest-axe (automated, runs in `sando-icon.a11y.test.ts`)
- VoiceOver (macOS Sequoia, Safari 18)
- NVDA (Windows 11, Firefox 130)

## Known limitations

None identified.
