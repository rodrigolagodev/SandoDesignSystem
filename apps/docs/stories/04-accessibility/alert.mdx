# Accessibility — sando-alert

> Source: [`packages/components/src/components/alert/ACCESSIBILITY.md`](https://github.com/rodrigolagodev/SandoDesignSystem/blob/master/packages/components/src/components/alert/ACCESSIBILITY.md)

## ARIA pattern

Follows the [WAI-ARIA Live Region](https://www.w3.org/WAI/ARIA/apg/patterns/alert/) pattern.

<table>
  <thead>
    <tr>
      <th>Attribute</th>
      <th>Value</th>
      <th>Purpose</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`role`</td>
      <td>`status` (default) / `alert`</td>
      <td>`status` = polite live region; `alert` = assertive live region for urgent messages</td>
    </tr>
    <tr>
      <td>`aria-live`</td>
      <td>`polite` (default) / `assertive`</td>
      <td>Controls urgency of screen reader announcement</td>
    </tr>
    <tr>
      <td>`aria-atomic`</td>
      <td>`true`</td>
      <td>Announces the entire container as a unit</td>
    </tr>
    <tr>
      <td>`aria-label`</td>
      <td>`"Dismiss alert"`</td>
      <td>On the close button when `dismissible=true`</td>
    </tr>
    <tr>
      <td>`aria-hidden`</td>
      <td>`true`</td>
      <td>On the icon wrapper — icon is decorative</td>
    </tr>
  </tbody>
</table>

When `role="none"` is set, the container has no `role` or `aria-live` attributes (purely decorative use).

## Screen reader behavior

For `role="status"` (default): Content is read politely at the next opportunity.

For `role="alert"`: Content interrupts current speech immediately.

## Color contrast

Verified via `jest-axe` against the `original` flavor in light and dark modes. All pairs meet WCAG 2.1 AA.

## Tested with

- jest-axe (automated, runs in `sando-alert.a11y.test.ts`)
- VoiceOver (macOS Sequoia, Safari 18)
- NVDA (Windows 11, Firefox 130)

## Known limitations

JSDOM does not fire `animationend` events; dismiss animation tests simulate the event manually. Real browser behavior is tested manually.
