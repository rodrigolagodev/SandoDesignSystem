# Accessibility — sando-card

> Source: [`packages/components/src/components/card/ACCESSIBILITY.md`](https://github.com/rodrigolagodev/SandoDesignSystem/blob/master/packages/components/src/components/card/ACCESSIBILITY.md)

## ARIA pattern

Follows the [WAI-ARIA Article](https://www.w3.org/WAI/ARIA/apg/patterns/) pattern for content containers.

<table>
  <thead>
    <tr>
      <th>Attribute</th>
      <th>Value</th>
      <th>Purpose</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`aria-labelledby`</td>
      <td>Heading element ID</td>
      <td>Associates the card heading as its accessible name</td>
    </tr>
    <tr>
      <td>`aria-label`</td>
      <td>Optional override</td>
      <td>Replaces `aria-labelledby` when set</td>
    </tr>
    <tr>
      <td>`aria-disabled`</td>
      <td>`"true"` / `"false"`</td>
      <td>Marks the card as non-interactive without removing from browse mode</td>
    </tr>
    <tr>
      <td>`aria-busy`</td>
      <td>`"true"` / `"false"`</td>
      <td>Set during loading skeleton state</td>
    </tr>
  </tbody>
</table>

The `heading-level` property controls the rendered heading element (`h1`–`h6`) for correct document outline.

## Keyboard map

<table>
  <thead>
    <tr>
      <th>Key</th>
      <th>Behavior</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Tab</td>
      <td>Move focus to the card (when `clickable=true`) or to interactive children</td>
    </tr>
    <tr>
      <td>Shift+Tab</td>
      <td>Move focus backward</td>
    </tr>
    <tr>
      <td>Enter / Space</td>
      <td>Activate the card (when `clickable=true`)</td>
    </tr>
  </tbody>
</table>

Non-clickable cards are not keyboard focusable themselves.

## Screen reader behavior

Clickable card: "Card title, article" or "Card title, link". Loading state: `aria-busy="true"` signals content is loading.

## Color contrast

Verified via `jest-axe` against the `original` flavor in light and dark modes. Disabled state uses intentional reduced contrast.

## Tested with

- jest-axe (automated, runs in `sando-card.a11y.test.ts`)
- VoiceOver (macOS Sequoia, Safari 18)
- NVDA (Windows 11, Firefox 130)

## Known limitations

None identified.
