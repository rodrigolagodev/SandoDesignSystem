# Accessibility — sando-tooltip

> Source: [`packages/components/src/components/tooltip/ACCESSIBILITY.md`](https://github.com/rodrigolagodev/SandoDesignSystem/blob/master/packages/components/src/components/tooltip/ACCESSIBILITY.md)

## ARIA pattern

Follows the [WAI-ARIA Tooltip](https://www.w3.org/WAI/ARIA/apg/patterns/tooltip/) pattern.

<table>
  <thead>
    <tr>
      <th>Attribute</th>
      <th>Value</th>
      <th>Purpose</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`role`</td>
      <td>`tooltip`</td>
      <td>Marks the tooltip bubble as a tooltip</td>
    </tr>
    <tr>
      <td>`aria-hidden`</td>
      <td>`"true"` (closed) / `"false"` (open)</td>
      <td>Hides the tooltip from AT when not visible</td>
    </tr>
    <tr>
      <td>`aria-describedby`</td>
      <td>Tooltip element ID</td>
      <td>Injected onto the slotted trigger element to associate the tooltip</td>
    </tr>
  </tbody>
</table>

Injects `aria-describedby` onto the first slotted trigger element. Implements WCAG 1.4.13 (Content on Hover or Focus).

    </tr>

</table>

## Screen reader behavior

When trigger receives focus: "Delete item, button. Permanently removes this record." (button text + tooltip via `aria-describedby`). The tooltip content is announced as supplementary description.

## Color contrast

Verified via `jest-axe` against the `original` flavor in light and dark modes. All pairs meet WCAG 2.1 AA.

## Tested with

- jest-axe (automated, runs in `sando-tooltip.a11y.test.ts`)
- VoiceOver (macOS Sequoia, Safari 18)
- NVDA (Windows 11, Firefox 130)

## Known limitations

None identified.
