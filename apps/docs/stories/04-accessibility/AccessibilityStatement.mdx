# Accessibility Statement

Sando Design System is committed to meeting **WCAG 2.1 Level AA** standards across all components. Every component ships with automated accessibility tests (`jest-axe`), manual keyboard and screen reader testing, and a colocated `ACCESSIBILITY.md` file documenting its ARIA pattern, keyboard map, and known limitations.

## Our Commitment

- **WCAG 2.1 AA**: All components meet or exceed Level AA criteria
- **Automated testing**: `jest-axe` runs on every component in CI via `pnpm test`
- **Manual testing**: VoiceOver (macOS Sequoia, Safari 18) and NVDA (Windows 11, Firefox 130)
- **Keyboard accessibility**: Every interactive component is fully operable without a mouse
- **Color contrast**: OKLCH color space ensures perceptual uniformity across all flavors

## Reporting an Accessibility Issue

If you find an accessibility issue in any Sando component, please open an issue on GitHub and add the `a11y` label. Include:

1. Which component and which state/variant exhibits the issue
2. Which assistive technology you tested with (screen reader, browser, OS version)
3. The WCAG criterion that is violated
4. Steps to reproduce

## Testing Methodology

### Automated Testing

Every component has a `sando-<name>.a11y.test.ts` file that runs as part of `pnpm test`. These tests use:

- **`jest-axe`**: Runs `axe-core` rules against the component in all relevant states (default, disabled, error, loading, etc.)
- **`@open-wc/testing`**: Renders components in a real DOM environment (JSDOM)
- **Color contrast**: `axe-core` validates color contrast ratios against WCAG 2.1 AA thresholds

### Manual Testing

In addition to automated tests, components are manually tested with:

- **VoiceOver + Safari** (macOS Sequoia): Focus announcement, state changes, group navigation
- **NVDA + Firefox** (Windows 11): Form field associations, live region announcements, keyboard patterns

### Test Scope

Tests cover:
1. Default state — no violations
2. All relevant variants (size, appearance, status)
3. Interactive states (checked, selected, expanded, loading, disabled, error)
4. All supported flavors (original, strawberry, chocolate)
