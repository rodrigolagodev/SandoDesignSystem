# Accessibility — sando-textarea

> Source: [`packages/components/src/components/textarea/ACCESSIBILITY.md`](https://github.com/rodrigolagodev/SandoDesignSystem/blob/master/packages/components/src/components/textarea/ACCESSIBILITY.md)

## ARIA pattern

Follows the [WAI-ARIA Textbox (multiline)](https://www.w3.org/WAI/ARIA/apg/patterns/textbox/) pattern via native `<textarea>`.

<table>
  <thead>
    <tr>
      <th>Attribute</th>
      <th>Value</th>
      <th>Purpose</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`aria-invalid`</td>
      <td>`"true"` / `"false"`</td>
      <td>Communicates validation error state</td>
    </tr>
    <tr>
      <td>`aria-required`</td>
      <td>`"true"` / `"false"`</td>
      <td>Communicates required state</td>
    </tr>
    <tr>
      <td>`aria-describedby`</td>
      <td>Helper/error element ID</td>
      <td>Links helper or error text to the textarea</td>
    </tr>
  </tbody>
</table>

Label association uses a `<label>` element linked via `for`/`id`.

## Screen reader behavior

Default: "Comment, text area" (VoiceOver) / "Comment multi-line edit" (NVDA). With helper: "Comment, text area. Maximum 500 characters." Error: "Comment, invalid data, text area. Please enter at least 10 characters."

## Color contrast

Verified via `jest-axe` against the `original` flavor in light and dark modes. Disabled state uses intentional reduced contrast.

## Tested with

- jest-axe (automated, runs in `sando-textarea.a11y.test.ts`)
- VoiceOver (macOS Sequoia, Safari 18)
- NVDA (Windows 11, Firefox 130)

## Known limitations

None identified.
