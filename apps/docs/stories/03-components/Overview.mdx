## Usage

All components are Web Components and work with any framework:

```bash
npm install @sando/components @sando/tokens
```

```js
// Import tokens
import "@sando/tokens/css";

// Import components you need
import "@sando/components/button";
import "@sando/components/input";
```

```html
<sando-button variant="solid">Click me</sando-button>
<sando-input label="Email" type="email"></sando-input>
```

## Framework Integration

### Vanilla JS / HTML

```html
<script type="module">
  import "@sando/components/button";
</script>

<sando-button>Hello</sando-button>
```

### React

```jsx
import "@sando/components/button";

function App() {
  return <sando-button onClick={handleClick}>Hello</sando-button>;
}
```

### Vue

```vue
<template>
  <sando-button @click="handleClick">Hello</sando-button>
</template>

<script setup>
import "@sando/components/button";
</script>
```

## Component Features

Every Sando component includes:

- **Accessibility** — WCAG 2.1 AA compliant with proper ARIA
- **Theming** — Supports all flavors via the `flavor` attribute
- **Dark Mode** — Automatic adaptation via `prefers-color-scheme`
- **TypeScript** — Full type definitions included
- **Documentation** — Stories with examples and API docs
- **Tests** — Unit and accessibility testing

## Component Anatomy

Components follow the **Recipe** token layer, consuming semantic **Flavor** tokens:

```css
/* Component uses recipe tokens */
.sando-button {
  background: var(--sando-button-solid-bg);
  color: var(--sando-button-solid-text);
  padding: var(--sando-button-padding-y) var(--sando-button-padding-x);
  border-radius: var(--sando-button-border-radius);
}
```

This architecture enables theming by simply switching flavors — components automatically update.
