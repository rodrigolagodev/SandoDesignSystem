# Spacing Roles

Semantic spacing tokens that define layout and component spacing relationships. These tokens reference primitive spacing values from the Ingredients layer and give them contextual meaning.

## Token Naming Convention

All flavor spacing tokens follow this pattern:

```
--sando-spacing-[category]-[role]
```

Examples:
- `--sando-spacing-component-padding`
- `--sando-spacing-layout-section`
- `--sando-spacing-content-gap`

## Component Spacing

Tokens for internal component spacing (padding, gaps within components):

| Token | Purpose | Default Value | Usage |
|-------|---------|---------------|-------|
| `--sando-spacing-component-padding` | Default component padding | `var(--sando-space-4)` (1rem) | Button padding, input padding |
| `--sando-spacing-component-padding-sm` | Small component padding | `var(--sando-space-2)` (0.5rem) | Compact buttons, dense inputs |
| `--sando-spacing-component-padding-lg` | Large component padding | `var(--sando-space-6)` (1.5rem) | Spacious buttons, large inputs |
| `--sando-spacing-component-gap` | Gap between component elements | `var(--sando-space-3)` (0.75rem) | Icon and text spacing in buttons |
| `--sando-spacing-component-gap-sm` | Small gap | `var(--sando-space-2)` (0.5rem) | Tight element spacing |
| `--sando-spacing-component-gap-lg` | Large gap | `var(--sando-space-4)` (1rem) | Relaxed element spacing |

### Example Usage

```css
.button {
  padding: var(--sando-spacing-component-padding);
  gap: var(--sando-spacing-component-gap);
}

.button--small {
  padding: var(--sando-spacing-component-padding-sm);
  gap: var(--sando-spacing-component-gap-sm);
}

.button--large {
  padding: var(--sando-spacing-component-padding-lg);
  gap: var(--sando-spacing-component-gap-lg);
}
```

## Layout Spacing

Tokens for macro-level layout spacing (sections, containers):

| Token | Purpose | Default Value | Usage |
|-------|---------|---------------|-------|
| `--sando-spacing-layout-section` | Space between major sections | `var(--sando-space-16)` (4rem) | Section margins, page breaks |
| `--sando-spacing-layout-container` | Container padding | `var(--sando-space-6)` (1.5rem) | Main container padding |
| `--sando-spacing-layout-gutter` | Grid gutter | `var(--sando-space-4)` (1rem) | Column gaps in grids |
| `--sando-spacing-layout-stack` | Vertical stacking space | `var(--sando-space-8)` (2rem) | Vertical rhythm between blocks |
| `--sando-spacing-layout-inline` | Horizontal inline space | `var(--sando-space-4)` (1rem) | Horizontal spacing between items |

### Example Usage

```css
.section {
  margin-block: var(--sando-spacing-layout-section);
}

.container {
  padding-inline: var(--sando-spacing-layout-container);
}

.grid {
  gap: var(--sando-spacing-layout-gutter);
}

.stack > * + * {
  margin-block-start: var(--sando-spacing-layout-stack);
}
```

## Content Spacing

Tokens for content-specific spacing (typography, media):

| Token | Purpose | Default Value | Usage |
|-------|---------|---------------|-------|
| `--sando-spacing-content-paragraph` | Space between paragraphs | `var(--sando-space-4)` (1rem) | Paragraph margins |
| `--sando-spacing-content-heading` | Space after headings | `var(--sando-space-6)` (1.5rem) | Heading bottom margin |
| `--sando-spacing-content-list` | Space between list items | `var(--sando-space-3)` (0.75rem) | `<li>` margins |
| `--sando-spacing-content-gap` | General content gap | `var(--sando-space-4)` (1rem) | Flow content spacing |
| `--sando-spacing-content-media` | Space around media | `var(--sando-space-6)` (1.5rem) | Image/video margins |

### Example Usage

```css
p {
  margin-block-end: var(--sando-spacing-content-paragraph);
}

h1, h2, h3 {
  margin-block-end: var(--sando-spacing-content-heading);
}

li + li {
  margin-block-start: var(--sando-spacing-content-list);
}

img, video {
  margin-block: var(--sando-spacing-content-media);
}
```

## Form Spacing

Tokens for form-specific spacing relationships:

| Token | Purpose | Default Value | Usage |
|-------|---------|---------------|-------|
| `--sando-spacing-form-field-gap` | Space between form fields | `var(--sando-space-4)` (1rem) | Vertical spacing between inputs |
| `--sando-spacing-form-label-gap` | Space between label and input | `var(--sando-space-2)` (0.5rem) | Label-to-input spacing |
| `--sando-spacing-form-helper-gap` | Space between input and helper text | `var(--sando-space-2)` (0.5rem) | Helper text spacing |
| `--sando-spacing-form-action-gap` | Space before form actions | `var(--sando-space-6)` (1.5rem) | Submit button top margin |

### Example Usage

```css
.form-field + .form-field {
  margin-block-start: var(--sando-spacing-form-field-gap);
}

.label {
  margin-block-end: var(--sando-spacing-form-label-gap);
}

.helper-text {
  margin-block-start: var(--sando-spacing-form-helper-gap);
}

.form-actions {
  margin-block-start: var(--sando-spacing-form-action-gap);
}
```

## Stack Spacing Utility

Sando provides a stack utility pattern that uses spacing roles:

```css
/* Default stack (1rem / 16px gap) */
.stack {
  display: flex;
  flex-direction: column;
  gap: var(--sando-spacing-content-gap);
}

/* Small stack (0.5rem / 8px gap) */
.stack--sm {
  gap: var(--sando-space-2);
}

/* Large stack (2rem / 32px gap) */
.stack--lg {
  gap: var(--sando-spacing-layout-stack);
}

/* Extra large stack (4rem / 64px gap) */
.stack--xl {
  gap: var(--sando-spacing-layout-section);
}
```

### Example Usage

```html
<div class="stack">
  <h1>Heading</h1>
  <p>Paragraph with consistent spacing</p>
  <p>Another paragraph</p>
</div>

<div class="stack stack--lg">
  <section>Section 1</section>
  <section>Section 2</section>
</div>
```

## Responsive Spacing

While spacing tokens have fixed values, you can create responsive spacing using CSS:

```css
.section {
  /* Mobile: smaller section spacing */
  margin-block: var(--sando-space-8);
}

@media (min-width: 768px) {
  .section {
    /* Tablet+: use semantic layout spacing */
    margin-block: var(--sando-spacing-layout-section);
  }
}
```

## Logical Properties

All spacing roles are designed to work with **CSS logical properties** for RTL support:

```css
/* ‚úÖ GOOD: Logical properties (RTL-friendly) */
.container {
  padding-inline: var(--sando-spacing-layout-container);
  margin-block-end: var(--sando-spacing-layout-stack);
}

/* ‚ùå BAD: Physical properties (not RTL-friendly) */
.container {
  padding-left: var(--sando-spacing-layout-container);
  padding-right: var(--sando-spacing-layout-container);
  margin-bottom: var(--sando-spacing-layout-stack);
}
```

**Logical property reference**:
- `padding-inline` = `padding-left` + `padding-right` (horizontal)
- `padding-block` = `padding-top` + `padding-bottom` (vertical)
- `margin-inline-start` = `margin-left` in LTR, `margin-right` in RTL
- `margin-block-end` = `margin-bottom` in horizontal writing modes

## Customizing Spacing Roles

You can customize spacing roles per flavor:

```css
/* Default spacing */
:root {
  --sando-spacing-component-padding: var(--sando-space-4);
}

/* Custom "compact" flavor with tighter spacing */
[flavor="compact"] {
  --sando-spacing-component-padding: var(--sando-space-2);
  --sando-spacing-layout-section: var(--sando-space-12);
  --sando-spacing-content-gap: var(--sando-space-3);
}

/* Custom "spacious" flavor with looser spacing */
[flavor="spacious"] {
  --sando-spacing-component-padding: var(--sando-space-6);
  --sando-spacing-layout-section: var(--sando-space-20);
  --sando-spacing-content-gap: var(--sando-space-6);
}
```

## Best Practices

### ‚úÖ DO

- Use semantic spacing roles for component and layout spacing
- Combine with logical properties for RTL support
- Create custom flavors to adjust spacing density
- Use the spacing scale consistently (4px base units)

### ‚ùå DON'T

- Use raw pixel values (`padding: 16px;`)
- Skip the Flavors layer and use Ingredients directly in components
- Mix spacing systems (stick to Sando's scale)
- Use physical properties (`margin-left`) instead of logical (`margin-inline-start`)
