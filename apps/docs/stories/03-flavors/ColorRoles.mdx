# Color Roles

Semantic color tokens that give meaning to primitive color values. These tokens automatically adapt to light mode, dark mode, and high contrast based on user preferences.

## Token Naming Convention

All flavor color tokens follow this pattern:

```
--sando-color-[category]-[role]-[state?]
```

Examples:
- `--sando-color-background-base`
- `--sando-color-text-body`
- `--sando-color-action-solid-background-hover`

## Background Colors

Tokens for surface and background colors:

| Token | Purpose | Usage |
|-------|---------|-------|
| `--sando-color-background-base` | Main page background | Body, canvas, main content area |
| `--sando-color-background-surface` | Secondary background | Cards, panels, elevated surfaces |
| `--sando-color-background-raised` | Highest elevation background | Modals, popovers, tooltips |
| `--sando-color-background-overlay` | Overlay backdrop | Behind modals and dialogs |
| `--sando-color-background-emphasis` | Emphasized background | Highlighted sections, selected items |

### Example Usage

```css
.page {
  background: var(--sando-color-background-base);
}

.card {
  background: var(--sando-color-background-surface);
}

.modal {
  background: var(--sando-color-background-raised);
}

.modal-backdrop {
  background: var(--sando-color-background-overlay);
  opacity: 0.5;
}
```

## Text Colors

Tokens for typography and content:

| Token | Purpose | Usage |
|-------|---------|-------|
| `--sando-color-text-heading` | Heading text | h1, h2, h3, h4, h5, h6 |
| `--sando-color-text-body` | Body text | Paragraphs, most readable content |
| `--sando-color-text-caption` | Caption/secondary text | Captions, helper text, metadata |
| `--sando-color-text-muted` | De-emphasized text | Disabled, placeholder, tertiary |
| `--sando-color-text-on-solid` | Text on solid backgrounds | White/light text on dark buttons |

| `--sando-color-text-link-hover` | Hovered link |

## Focus Colors

Tokens for focus indicators (WCAG 2.1 AA compliant):

| Token | Purpose | Usage |
|-------|---------|-------|
| `--sando-color-focus-ring` | Focus ring outline | Focused interactive elements |
| `--sando-color-focus-ring-offset` | Focus ring offset/background | Space between element and ring |
| `--sando-color-focus-background` | Focus background | Highlighted focused element |
| `--sando-color-focus-text` | Focus text | Text color when focused |

### Example Usage

```css
.button:focus-visible {
  outline: 2px solid var(--sando-color-focus-ring);
  outline-offset: 2px;
  background: var(--sando-color-focus-background);
}

.input:focus {
  outline: 2px solid var(--sando-color-focus-ring);
  outline-offset: -1px;
}
```

## Automatic Dark Mode

All color roles automatically adapt to dark mode via `@media (prefers-color-scheme: dark)`:

```css
/* Light mode (default) */
:root {
  --sando-color-background-base: var(--sando-color-neutralWarm-100);
  --sando-color-text-body: var(--sando-color-neutralWarm-800);
}

/* Dark mode (automatic) */
@media (prefers-color-scheme: dark) {
  :root {
    --sando-color-background-base: var(--sando-color-neutralWarm-900);
    --sando-color-text-body: var(--sando-color-neutralWarm-200);
  }
}
```

**You don't need to write any mode-switching logic!** Just use the semantic tokens.

## WCAG Compliance

All color role combinations meet WCAG 2.1 AA standards:
- Normal text: 4.5:1 contrast ratio
- Large text: 3:1 contrast ratio
- UI components: 3:1 contrast ratio

When using color roles as documented, your UI will be accessible by default.
