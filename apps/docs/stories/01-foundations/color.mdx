# Color

The Sando Design System provides **8 scientifically-designed color palettes** + **3 neutral variants**, all built using the **OKLCH color space** for perceptual uniformity and guaranteed accessibility (WCAG AA).

## Color Philosophy

Sando's color system is **opinionated and curated**:

- ✅ **8 Pre-designed Palettes**: Choose the palette that matches your brand personality
- ✅ **OKLCH Color Space**: Perceptually uniform, predictable contrast
- ✅ **WCAG AA Guaranteed**: All palettes pass accessibility standards
- ✅ **Instant Start**: Pick a palette and begin coding in 2 minutes

**Three-layer architecture:**

1. **Ingredients** (this page) - 8 curated color palettes + 3 neutral variants
2. **Flavors** - Map palettes to semantic roles (e.g., `action.solid.background`)
3. **Recipes** - Component-specific tokens (e.g., `button.solid.backgroundColor`)

Learn more about our [Color Philosophy](/?path=/docs/color-philosophy--docs).

## Usage Guidelines

### Do's

- ✅ Use Ingredients tokens directly in rare cases when building new Flavor tokens
- ✅ Reference brand colors for primary actions and brand moments
- ✅ Use neutral colors for text, backgrounds, and UI structure
- ✅ Apply state colors consistently across all feedback scenarios

### Don'ts

- ❌ Don't use Ingredients tokens directly in components (use Recipes instead)
- ❌ Don't mix color systems (stay within the design system)
- ❌ Don't use hardcoded color values in your code
- ❌ Don't use state colors for decorative purposes

---

## Accessibility

All color combinations in our Flavors layer meet **WCAG 2.1 AA** standards for color contrast:
- **Text**: Minimum 4.5:1 contrast ratio
- **Large text**: Minimum 3:1 contrast ratio
- **UI components**: Minimum 3:1 contrast ratio

The color system is designed to work seamlessly in both light and dark modes.
