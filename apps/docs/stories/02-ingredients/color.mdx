# Color

The Sando Design System provides **6 brand color palettes** + **3 neutral variants**, all scientifically generated using **OKLCH color space** for perceptual uniformity, then compiled to **HSL** for maximum browser compatibility.

## Color Philosophy

Sando's color system is **opinionated and curated**:

- ‚úÖ **6 Brand Palettes**: Orange, Blue, Green, Red, Purple, Pink
- ‚úÖ **3 Neutral Variants**: Neutral (gray), Neutral Cool (blue-gray), Neutral Warm (tan)
- ‚úÖ **OKLCH-Generated**: Perceptually uniform scales, compiled to HSL
- ‚úÖ **WCAG AA Guaranteed**: All palettes pass accessibility standards
- ‚úÖ **11 Shades**: From 50 (lightest) to 950 (darkest)

**Three-layer architecture:**

1. **Ingredients** (this page) - 6 brand palettes + 3 neutral variants
2. **Flavors** - Map palettes to semantic roles (e.g., `action.solid.background`)
3. **Recipes** - Component-specific tokens (e.g., `button.solid.backgroundColor`)

## Usage Guidelines

### Do's

- ‚úÖ Use Ingredients tokens directly in rare cases when building new Flavor tokens
- ‚úÖ Reference brand colors for primary actions and brand moments
- ‚úÖ Use neutral colors for text, backgrounds, and UI structure
- ‚úÖ Apply state colors consistently across all feedback scenarios

### Don'ts

- ‚ùå Don't use Ingredients tokens directly in components (use Recipes instead)
- ‚ùå Don't mix color systems (stay within the design system)
- ‚ùå Don't use hardcoded color values in your code
- ‚ùå Don't use state colors for decorative purposes

---

## Accessibility

All color combinations in our Flavors layer meet **WCAG 2.1 AA** standards for color contrast:
- **Text**: Minimum 4.5:1 contrast ratio
- **Large text**: Minimum 3:1 contrast ratio
- **UI components**: Minimum 3:1 contrast ratio

The color system is designed to work seamlessly in both light and dark modes.
