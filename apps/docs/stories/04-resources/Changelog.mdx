# Changelog

All notable changes to the Sando Design System are documented here. We follow [Semantic Versioning](https://semver.org/) for releases.

---

## Version Format

```
MAJOR.MINOR.PATCH

MAJOR - Breaking changes that require migration
MINOR - New features, backwards compatible
PATCH - Bug fixes, backwards compatible
```

---

#### Highlights

This is the initial preview release of the Sando Design System, establishing the foundation for a complete design system built on Web Components.

#### Added

**Token System**

- Three-layer token architecture (Ingredients, Flavors, Recipes)
- 8 brand color palettes (Orange, Yellow, Brown, Red, Pink, Purple, Blue, Green)
- 3 neutral color variants (Neutral, Neutral Warm, Neutral Cool)
- Complete spacing scale (4px base unit, 0-64 steps)
- Typography scale (10px-48px)
- Border radius and width tokens
- Elevation system (6 levels)
- Animation duration and easing tokens
- Opacity scale (0-100%)
- Z-index semantic scale

**Flavor Themes**

- Original (Orange) - default theme
- Strawberry (Red)
- Tonkatsu (Brown)
- Kiwi (Green)
- Egg Salad (Yellow)

**Components**

- `sando-button` - Button component with solid, outline, and ghost variants
- `sando-icon` - Icon component with Lucide icon set
- `sando-input` - Text input component with validation states
- `sando-form-group` - Form layout component (beta)

**Documentation**

- Storybook documentation site
- Token reference pages
- Component usage examples
- Accessibility guidelines

#### Known Limitations

- Dark mode is implemented but may have minor contrast issues
- High contrast mode support is partial
- Some neutral color palettes are still in development

## Release Process

1. **Alpha** (`0.x.x`) - Active development, API may change
2. **Beta** (`0.x.x-beta.x`) - Feature complete, testing phase
3. **Release Candidate** (`0.x.x-rc.x`) - Final testing before release
4. **Stable** (`1.x.x+`) - Production ready, semantic versioning
