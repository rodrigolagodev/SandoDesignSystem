# Input Tokens

Component-specific tokens for `<sando-input>`. These Recipe tokens ensure form inputs are consistent, accessible, and theme-aware across all flavors.

## Token Reference

### Outline Variant (Default)

The standard input style with visible borders.

| Token | Purpose | Default Value |
|-------|---------|---------------|
| `--sando-input-outline-backgroundColor-default` | Default background | `var(--sando-color-background-raised)` |
| `--sando-input-outline-backgroundColor-disabled` | Disabled background | `var(--sando-color-background-emphasis)` |
| `--sando-input-outline-backgroundColor-focus` | Focused background | `var(--sando-color-background-raised)` |
| `--sando-input-outline-textColor-default` | Default text color | `var(--sando-color-text-body)` |
| `--sando-input-outline-textColor-disabled` | Disabled text color | `var(--sando-color-text-muted)` |
| `--sando-input-outline-textColor-placeholder` | Placeholder text | `var(--sando-color-text-caption)` |
| `--sando-input-outline-borderColor-default` | Default border | `var(--sando-color-border-default)` |
| `--sando-input-outline-borderColor-hover` | Hovered border | `var(--sando-color-border-emphasis)` |
| `--sando-input-outline-borderColor-focus` | Focused border | `var(--sando-color-border-emphasis)` |
| `--sando-input-outline-borderColor-error` | Error border | `var(--sando-color-state-error-border)` |
| `--sando-input-outline-borderColor-disabled` | Disabled border | `var(--sando-color-border-muted)` |

| `--sando-input-filled-backgroundColor-focus` | Focused background | `var(--sando-color-background-raised)` |
| `--sando-input-filled-backgroundColor-disabled` | Disabled background | `var(--sando-color-background-emphasis)` |
| `--sando-input-filled-textColor-default` | Default text color | `var(--sando-color-text-body)` |
| `--sando-input-filled-textColor-disabled` | Disabled text color | `var(--sando-color-text-muted)` |
| `--sando-input-filled-borderColor-focus` | Focused underline | `var(--sando-color-border-emphasis)` |
| `--sando-input-filled-borderColor-error` | Error underline | `var(--sando-color-state-error-border)` |

## Customization Examples

### Example 1: Custom Input Colors

```css
.input--custom {
  --sando-input-outline-borderColor-default: var(--sando-color-purple-400);
  --sando-input-outline-borderColor-focus: var(--sando-color-purple-600);
  --sando-input-outline-backgroundColor-default: var(--sando-color-purple-50);
}
```

```html
<sando-input class="input--custom" label="Custom Input"></sando-input>
```

### Example 2: Success State

```css
.input--success {
  --sando-input-outline-borderColor-default: var(--sando-color-state-success-border);
  --sando-input-outline-borderColor-focus: var(--sando-color-state-success-600);
}
```

```html
<sando-input class="input--success" label="Valid Email"></sando-input>
```

### Example 3: Per-Flavor Input Styling

```css
/* Forest flavor uses green accents */
[flavor="forest"] {
  --sando-input-outline-borderColor-focus: var(--sando-color-green-600);
  --sando-input-outline-backgroundColor-focus: var(--sando-color-green-50);
}
```

```html
<div flavor="forest">
  <sando-input label="Search"></sando-input>
</div>
```

### Example 4: Compact Input Variant

```css
[flavor="compact"] {
  --sando-input-paddingInline: var(--sando-space-3);
  --sando-input-paddingBlock: var(--sando-space-2);
  --sando-input-fontSize: var(--sando-font-size-300);
}
```

## State Management

Input tokens handle multiple states automatically:

| State | Border Token Used |
|-------|-------------------|
| Default | `--sando-input-outline-borderColor-default` |
| Hover | `--sando-input-outline-borderColor-hover` |
| Focus | `--sando-input-outline-borderColor-focus` |
| Error | `--sando-input-outline-borderColor-error` |
| Disabled | `--sando-input-outline-borderColor-disabled` |

States combine appropriately (e.g., error + focus shows error border).

## Accessibility Notes

All input tokens maintain WCAG 2.1 AA requirements:
- **Border contrast**: 3:1 against background
- **Text contrast**: 4.5:1 against background
- **Placeholder contrast**: 4.5:1 (caption color)
- **Error text**: 4.5:1 with error icon
- **Focus indicator**: Visible focus ring via `--sando-color-focus-ring`

The input component automatically includes:
- Associated labels (aria-labelledby)
- Error messages (aria-describedby, aria-invalid)
- Required indicators (aria-required)
- Disabled state (disabled attribute)

## Form Integration

Input tokens work seamlessly with `<sando-form-group>`:

```html
<sando-form-group>
  <sando-input
    label="Email"
    type="email"
    required
    error="Please enter a valid email"
  ></sando-input>
</sando-form-group>
```

The form group component uses the same token system for consistent spacing and styling.
