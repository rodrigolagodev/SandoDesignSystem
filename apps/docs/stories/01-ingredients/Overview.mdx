# Ingredients (Primitives)

Ingredients are the raw, atomic design tokens that form the foundation of Sando Design System. Like quality ingredients in a gourmet sandwich, these primitives ensure consistency across your entire application.

## What Are Ingredients?

Ingredients are **context-agnostic primitive values**. They define the raw palette of options without prescribing how they should be used. Think of them as your design vocabulary.

## Usage Pattern

Ingredients are typically not used directly in components. Instead, they're referenced by **Flavors** (semantic tokens) which are then consumed by **Recipes** (component tokens).

```css
/* Ingredient (primitive) */
--sando-color-orange-500: hsl(25, 95%, 53%);

/* Flavor (semantic) - references ingredient */
--sando-color-action-solid-background: var(--sando-color-orange-500);

/* Recipe (component) - references flavor */
--sando-button-solid-bg: var(--sando-color-action-solid-background);
```

## Why This Architecture?

1. **Single Source of Truth** — Change an ingredient, and all dependent tokens update
2. **Easy Theming** — Swap flavors without touching component code
3. **Consistent Design** — Components automatically follow the system
4. **Maintainability** — Clear hierarchy of dependencies
