# Elevation

Elevation creates visual hierarchy and depth through carefully crafted shadows. The Sando Design System provides **6 elevation levels** built on **Ingredients tokens** (Layer 1).

## Elevation Scale

Our elevation system uses layered shadows to simulate depth and focus, helping users understand element hierarchy and interactive states.

## Usage Guidelines

### When to Use Elevation

- **elevation-0**: Flat elements, embedded content
- **elevation-100**: Subtle separation, hover states for flat elements
- **elevation-200**: Cards, buttons, raised content
- **elevation-300**: Dropdowns, popovers, floating menus
- **elevation-400**: Modal dialogs, sheets, overlays
- **elevation-500**: Tooltips, toast notifications, top-level UI

### Do's

- ✅ Use elevation to establish visual hierarchy
- ✅ Increase elevation on hover for interactive elements
- ✅ Keep elevation consistent for similar components
- ✅ Use higher elevations sparingly for critical UI

### Don'ts

- ❌ Don't use too many elevation levels in one view
- ❌ Don't use elevation without semantic meaning
- ❌ Don't mix elevation systems
- ❌ Don't use elevation as decoration only

---

## Dark Mode Considerations

Elevation shadows work differently in dark mode:

- **Light mode**: Shadows are dark (rgba(0,0,0))
- **Dark mode**: May need lighter shadows or borders for definition
- **Future enhancement**: Dynamic shadow colors based on flavor mode

## Accessibility

- Elevation alone should not convey critical information
- Combine with color, text, or icons for important states
- Ensure sufficient contrast for shadow visibility
- Consider motion-reduced preferences for elevation transitions
