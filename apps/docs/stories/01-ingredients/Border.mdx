# Border

Border properties control the visual edges and roundness of UI elements. The Sando Design System provides **border radius** and **border width** tokens built on **Ingredients tokens** (Layer 1).

## Border Radius

Our border radius scale ranges from sharp corners to fully circular elements, providing flexibility for different design needs.

## Usage Guidelines

### Border Radius

**When to Use:**

- **radius-0**: Geometric, technical interfaces
- **radius-50-100**: Subtle softness, modern look
- **radius-200**: Default for most UI elements
- **radius-300-400**: Friendly, approachable feel
- **radius-500-600**: Distinctive, playful elements
- **radius-circle**: Icons, avatars, badges
- **radius-full**: Pills, tags, infinite buttons

### Border Width

**When to Use:**

- **width-0**: Borderless elements
- **width-50**: Hairline separators, subtle outlines
- **width-100**: Default borders for inputs, cards
- **width-150-200**: Emphasized borders, selected states
- **width-300-400**: Focus indicators, decorative borders

### Do's

- ✅ Use consistent border radius within component families
- ✅ Apply larger radii to larger elements
- ✅ Use border-full for pill-shaped buttons
- ✅ Use hairline borders for subtle separation

### Don'ts

- ❌ Don't mix too many different radii in one interface
- ❌ Don't use very large radii on small elements
- ❌ Don't use arbitrary border values outside the scale
- ❌ Don't use thick borders without purpose

---

## Accessibility

- Border should not be the only indicator of focus or state
- Combine with color changes or shadows for better visibility
- Ensure sufficient contrast between border and background (3:1 minimum)
- Consider increased border width for focus states
