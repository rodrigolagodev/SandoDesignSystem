# Typography

The Sando Design System typography foundation provides a comprehensive type scale, font families, weights, and spacing controls built on **Ingredients tokens** (Layer 1).

---

    intentional and follows industry standards (IBM Carbon, Material Design,
    etc.). Tokens define <code>font-family</code> stacks with system fallbacks
    that work without loading external fonts.
  </span>
</div>

### Loading Google Fonts (Recommended)

To use the full range of flavor-specific typography, add this snippet to your HTML `<head>`:

<div className="demo-code-dark">
  <pre>{`<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link href="https://fonts.googleapis.com/css2?family=Cormorant+Garamond:ital,wght@0,400;0,500;0,600;0,700;1,400&family=DM+Sans:wght@400;500;600;700&family=Fira+Code:wght@400;500&family=Inter:wght@400;500;600;700&family=JetBrains+Mono:wght@400;500&family=Literata:wght@400;500;600;700&family=Merriweather:wght@400;700&family=Newsreader:wght@400;500;600;700&family=Plus+Jakarta+Sans:wght@400;500;600;700&family=Space+Grotesk:wght@400;500;600;700&display=swap" rel="stylesheet" />`}</pre>
</div>

## Usage Guidelines

### Do's

- Use the type scale consistently across all components
- Use sans-serif for UI text, monospace for code
- Apply appropriate line heights for content type
- Use font weights to create hierarchy

### Don'ts

- Don't use arbitrary font sizes outside the scale
- Don't mix too many weights in a single interface
- Don't use tight line heights for body text
- Don't use excessive letter spacing on body text

---

## Accessibility

- Minimum font size for body text: `font-size-200` (14px)
- Line height for body text should be at least 1.5 (`lineHeight-150`)
- Ensure sufficient color contrast (see Color foundation)
- Use relative units (rem) for responsive scaling
