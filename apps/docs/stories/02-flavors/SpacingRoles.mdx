# Spacing Roles

Semantic spacing tokens that define layout and component spacing relationships. These tokens reference primitive spacing values from the Ingredients layer and give them contextual meaning.

## Token Naming Convention

All flavor spacing tokens follow this pattern:

```
--sando-spacing-[category]-[role]
```

Examples:

- `--sando-spacing-component-padding`
- `--sando-spacing-layout-section`
- `--sando-spacing-content-gap`

## Stack Spacing Utility

Sando provides a stack utility pattern that uses spacing roles:

```css
/* Default stack (1rem / 16px gap) */
.stack {
  display: flex;
  flex-direction: column;
  gap: var(--sando-spacing-content-gap);
}

/* Small stack (0.5rem / 8px gap) */
.stack--sm {
  gap: var(--sando-space-2);
}

/* Large stack (2rem / 32px gap) */
.stack--lg {
  gap: var(--sando-spacing-layout-stack);
}

/* Extra large stack (4rem / 64px gap) */
.stack--xl {
  gap: var(--sando-spacing-layout-section);
}
```

### Example Usage

```html
<div class="stack">
  <h1>Heading</h1>
  <p>Paragraph with consistent spacing</p>
  <p>Another paragraph</p>
</div>

<div class="stack stack--lg">
  <section>Section 1</section>
  <section>Section 2</section>
</div>
```

## Responsive Spacing

While spacing tokens have fixed values, you can create responsive spacing using CSS:

```css
.section {
  /* Mobile: smaller section spacing */
  margin-block: var(--sando-space-8);
}

@media (min-width: 768px) {
  .section {
    /* Tablet+: use semantic layout spacing */
    margin-block: var(--sando-spacing-layout-section);
  }
}
```

## Logical Properties

All spacing roles are designed to work with **CSS logical properties** for RTL support:

```css
/* ‚úÖ GOOD: Logical properties (RTL-friendly) */
.container {
  padding-inline: var(--sando-spacing-layout-container);
  margin-block-end: var(--sando-spacing-layout-stack);
}

/* ‚ùå BAD: Physical properties (not RTL-friendly) */
.container {
  padding-left: var(--sando-spacing-layout-container);
  padding-right: var(--sando-spacing-layout-container);
  margin-bottom: var(--sando-spacing-layout-stack);
}
```

**Logical property reference**:

- `padding-inline` = `padding-left` + `padding-right` (horizontal)
- `padding-block` = `padding-top` + `padding-bottom` (vertical)
- `margin-inline-start` = `margin-left` in LTR, `margin-right` in RTL
- `margin-block-end` = `margin-bottom` in horizontal writing modes

## Customizing Spacing Roles

You can customize spacing roles per flavor:

```css
/* Default spacing */
:root {
  --sando-spacing-component-padding: var(--sando-space-4);
}

/* Custom "compact" flavor with tighter spacing */
[flavor="compact"] {
  --sando-spacing-component-padding: var(--sando-space-2);
  --sando-spacing-layout-section: var(--sando-space-12);
  --sando-spacing-content-gap: var(--sando-space-3);
}

/* Custom "spacious" flavor with looser spacing */
[flavor="spacious"] {
  --sando-spacing-component-padding: var(--sando-space-6);
  --sando-spacing-layout-section: var(--sando-space-20);
  --sando-spacing-content-gap: var(--sando-space-6);
}
```

## Best Practices

### ‚úÖ DO

- Use semantic spacing roles for component and layout spacing
- Combine with logical properties for RTL support
- Create custom flavors to adjust spacing density
- Use the spacing scale consistently (4px base units)

### ‚ùå DON'T

- Use raw pixel values (`padding: 16px;`)
- Skip the Flavors layer and use Ingredients directly in components
- Mix spacing systems (stick to Sando's scale)
- Use physical properties (`margin-left`) instead of logical (`margin-inline-start`)
