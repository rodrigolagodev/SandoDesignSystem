# Color Roles

Semantic color tokens that give meaning to primitive color values. These tokens automatically adapt to light mode, dark mode, and high contrast based on user preferences.

## Token Naming Convention

All flavor color tokens follow this pattern:

```
--sando-color-[category]-[role]-[state?]
```

Examples:

- `--sando-color-background-base`
- `--sando-color-text-body`
- `--sando-color-action-solid-background-hover`

## Automatic Dark Mode

All color roles automatically adapt to dark mode via `@media (prefers-color-scheme: dark)`:

```css
/* Light mode (default) */
:root {
  --sando-color-background-base: var(--sando-color-neutralWarm-100);
  --sando-color-text-body: var(--sando-color-neutralWarm-800);
}

/* Dark mode (automatic) */
@media (prefers-color-scheme: dark) {
  :root {
    --sando-color-background-base: var(--sando-color-neutralWarm-900);
    --sando-color-text-body: var(--sando-color-neutralWarm-200);
  }
}
```

**You don't need to write any mode-switching logic!** Just use the semantic tokens.

## WCAG Compliance

All color role combinations meet WCAG 2.1 AA standards:

- Normal text: 4.5:1 contrast ratio
- Large text: 3:1 contrast ratio
- UI components: 3:1 contrast ratio

When using color roles as documented, your UI will be accessible by default.
